Programa de escritorio para Gestor de inventario

Video de demostración:
Un poco sobre el proyecto

Este proyecto se realizó con el fin de agilizar la gestión del inventario de un negocio de venta de productos cárnicos, especialmente pollo. El negocio manejaba el inventario de manera manual (en cuadernos fisicos). Con este programa se agilizó el registro de entradas y salidas de producto llevando conteo de inventario, así como las cuentas y ganancias de las ventas y compras.

Para el desarrollo se usó el lenguaje Java, aplicando el patron de diseño MVC o Modelo-Vista-Controlador, el diseño de la interfaz se hizo en Figma, se usó Java Swing para aplicar el diseño de figma dentro de Java. Todas las entradas, salidas y registros en general se almacenan en una base de datos de MySQL.

Principales caracteristicas o funciones del programa:
  • Agregar productos
  • Agregar clientes
  • Agregar proveedores
  • Realizar entradas de producto (compras)
  • Realizar salidas de producto (ventas)
  • Gestionar el inventario de los productos
  • Calcular las ganancias y gastos de compras y ventas por rango de fechas
  • Generar reportes en PDF por rango de fechas
Tecnologías usadas
Java
MySQL
MVC

Aplicación web para un SPA de mascotas

Video de demostración del prototipado en Figma:
Un poco sobre el proyecto

Proyecto del ciclo 3 (de 4 ciclos) de Misiontic2022 en la Universidad Tecnológica de Pereira. El proyecto fue realizado dentro de un equipo de 5 personas en el que desempeñé el rol de líder o "Gestor del proyecto" organizando tareas, realizando historias de usuario, product backlog, entre otras... Además hice gran parte del diseño de la aplicación usando Figma.

El proyecto tiene como finalidad permitirle al dueño de un SPA de mascotas llevar un control de los servicios prestados a las mascotas de sus clientes. La aplicación tiene como funcionalidades agregar, eliminar, editar y visualizar clientes, servicios que presta el SPA y los servicios prestados a las mascotas de los clientes

En el desarrollo del Frontend se realizó con HTML, CSS, JavaScript y Bootstrap. Por el lado del Backend se usó Java con SpringBoot e Hibernate y la arquitectura de microservicios. Todo el proyecto fue realizado bajo la metodología SCRUM realizando la entrega de 4 Sprints.
